# 周末时间回顾 App

一个本地使用的周末时间回顾 MVP，用固定时间块和固定分类记录周末活动，并生成时间画像与规则化趋势总结。

## 如何运行

```bash
npm install
npm run dev
```

开发服务启动后，打开终端中显示的本地地址即可使用。

生产构建：

```bash
npm run build
```

## MVP 功能范围

- 通过月历选择周末日期，并以 `YYYY-MM-DD` 保存记录日期。
- 固定 6 个周末时间块：周六上午、周六下午、周六晚上、周日上午、周日下午、周日晚上。
- 同一天内支持多选时间块，提交后按每个时间块生成记录。
- 固定 9 个分类：休息、社交、运动、学习、娱乐、家务、出行、工作、陪伴家人。
- 支持新增、编辑、删除过往周末和本周末记录。
- 月历支持月份翻页，周末格子展示当天已有记录。
- 记录列表按日期和时间块倒序展示。
- 时间画像支持本月、最近三个月、本年、使用以来筛选。
- 时间画像使用饼状图展示分类占比。
- 趋势总结支持最近一个月、最近 3 个月、最近一年切换。
- 数据保存在浏览器 `localStorage`，不包含账号登录、云同步、备注、照片、心情等非 MVP 功能。

## 统计口径

本应用不按精确小时统计，而是沿用时间块权重：

- 每个 `日期 + 时间块` 计为 1 个时间块权重。
- 多选时间块会拆成多条记录分别参与统计。
- 如果同一个时间块内有多条记录，则这些记录平分该时间块权重。
- 分类占比 = 分类权重 / 当前筛选范围内的总时间块权重。

例如：同一个周六上午记录了“学习”和“运动”两件事，则学习和运动各获得 `0.5` 个时间块权重。

## 趋势总结规则

趋势总结完全使用规则判断，不使用 AI。

- 最近一个月、最近 3 个月、最近一年使用滚动周期统计。
- 每个趋势周期会与上一等长周期比较。
- 当前周期至少需要 2 个时间块权重，且当前周期与上一周期合计至少需要 3 个时间块权重，才会形成趋势。
- 记录不足时展示“暂时没有足够记录形成趋势”。
- 趋势会总结当前周期占比最高的分类、相比上一等长周期上升或下降最明显的分类，以及休息、社交、学习、工作等明显倾向。
